### 1.2 技术栈总览

| 层级 | 技术选型 | 说明 |
|-----|---------|------|
| **接入层** | Nginx + CDN | 反向代理、负载均衡、静态资源 |
| **前端** | Next.js 15 + React 19 | SSR、组件化、状态管理 |
| **后端** | FastAPI + Python 3.12 | 高性能API、异步处理 |
| **数据库** | PostgreSQL 16 | 关系型数据存储 |
| **缓存** | Redis 7 | 缓存、会话、消息队列 |
| **向量库** | Milvus | RAG知识库向量存储 |
| **对象存储** | MinIO | 文件、图片存储 |
| **AI服务** | OpenAI/Anthropic/Ollama | LLM模型调用 |
| **容器化** | Docker + Kubernetes | 容器编排、弹性伸缩 |
| **监控** | Prometheus + Grafana | 性能监控、可视化 |
| **日志** | ELK Stack | 日志收集、分析 |

## 三、数据流设计

### 3.1 对话数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                      对话数据流                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 用户发送消息                                                 │
│     ┌────────────────────────────────────────────────────────┐ │
│     │ POST /api/v1/chat/stream                              │ │
│     │ Body: {session_id, content, model, tools}             │ │
│     └────────────────────────────────────────────────────────┘ │
│                          │                                      │
│                          ▼                                      │
│  2. 创建用户消息                                                 │
│     ┌────────────────────────────────────────────────────────┐ │
│     │ INSERT INTO messages                                  │ │
│     │ (session_id, role='user', content)                    │ │
│     └────────────────────────────────────────────────────────┘ │
│                          │                                      │
│                          ▼                                      │
│  3. 构建对话上下文                                               │
│     ┌────────────────────────────────────────────────────────┐ │
│     │ SELECT messages FROM session ORDER BY created_at      │ │
│     │ LIMIT 10                                              │ │
│     │ → 构建 messages 数组                                   │ │
│     └────────────────────────────────────────────────────────┘ │
│                          │                                      │
│                          ▼                                      │
│  4. 调用LLM（流式）                                              │
│     ┌────────────────────────────────────────────────────────┐ │
│     │ POST https://api.openai.com/v1/chat/completions       │ │
│     │ Headers: Authorization: Bearer {api_key}              │ │
│     │ Body: {model, messages, stream=true}                  │ │
│     └────────────────────────────────────────────────────────┘ │
│                          │                                      │
│                          ▼ SSE chunks                           │
│  5. 流式返回                                                     │
│     ┌────────────────────────────────────────────────────────┐ │
│     │ SSE Event: data: {type: 'text', content: '...'}       │ │
│     │ SSE Event: data: {type: 'thinking', content: '...'}   │ │
│     │ SSE Event: data: {type: 'complete', metadata: {...}}  │ │
│     └────────────────────────────────────────────────────────┘ │
│                          │                                      │
│                          ▼                                      │
│  6. 更新AI消息                                                   │
│     ┌────────────────────────────────────────────────────────┐ │
│     │ UPDATE messages                                       │ │
│     │ SET content={...}, status='completed', metadata={...} │ │
│     │ WHERE id={message_id}                                 │ │
│     └────────────────────────────────────────────────────────┘ │
│                          │                                      │
│                          ▼                                      │
│  7. 更新会话                                                     │
│     ┌────────────────────────────────────────────────────────┐ │
│     │ UPDATE sessions                                       │ │
│     │ SET message_count+=2, last_message_at=NOW()          │ │
│     │ WHERE id={session_id}                                 │ │
│     └────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 GitHub Actions配置

```yaml
# .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '20'
      
      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.12'
      
      - name: Install Dependencies
        run: |
          cd frontend && npm ci
          cd backend && pip install -r requirements.txt
      
      - name: Run Tests
        run: |
          cd frontend && npm test
          cd backend && pytest
      
      - name: Security Scan
        uses: snyk/actions/node@master

  build:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Build Frontend Image
        run: |
          docker build -t ai-chat-frontend:${{ github.sha }} ./frontend
      
      - name: Build Backend Image
        run: |
          docker build -t ai-chat-backend:${{ github.sha }} ./backend
      
      - name: Push Images
        run: |
          docker push ai-chat-frontend:${{ github.sha }}
          docker push ai-chat-backend:${{ github.sha }}

  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to Kubernetes
        run: |
          kubectl set image deployment/frontend \
            frontend=ai-chat-frontend:${{ github.sha }}
          kubectl set image deployment/backend \
            backend=ai-chat-backend:${{ github.sha }}
          kubectl rollout status deployment/frontend
          kubectl rollout status deployment/backend
```
